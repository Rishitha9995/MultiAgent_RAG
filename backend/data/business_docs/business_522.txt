Design a data structure for storing a company's employee records.
For storing a company's employee records, one could use a database with a table named `employees` and define it with the following fields:

1. `employee_id`: This would be the primary key and uniquely identify each employee. It can be of datatype Integer and set to automatically increment.
2. `first_name`: This field would store the employee's first name. It can be of datatype String.
3. `last_name`: This field would store the employee's last name. It can be of datatype String.
4. `date_of_birth`: This field would store the employee's date of birth. It can be of datatype Date.
5. `hire_date`: This field would store the employee's hiring date. It can be of datatype Date.
6. `job_title`: This field would store the employee's job title. It can be of datatype String.
7. `department`: This field would store the employee's department. It can be of datatype String.
8. `salary`: This field would store the employee's salary. It can be of datatype Float.
9. `email`: This field would store the employee's email address. It can be of datatype String.
10. `phone_number`: This field would store the employee's phone number. It can be of datatype String.

This data structure allows for efficient storage and retrieval of employee records, and offers flexibility in terms of adding, updating, and deleting records. The records can easily be queried, sorted, and filtered based on various fields such as department, salary, job title, among others.
