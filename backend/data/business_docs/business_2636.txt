Design a database to keep track of an employee’s leave and holidays
A database to keep track of employee's leave and holidays information can contain the following tables and fields:

**Table 1: Employee**

- EmployeeID (Primary Key)
- FirstName
- LastName
- Department
- Position
- DateHired

**Table 2: LeaveType**

- LeaveTypeID (Primary Key)
- LeaveType (ex: Paid leave, Sick leave, Parental leave, etc.)
- Description

**Table 3: Leave**

- LeaveID (Primary Key)
- EmployeeID (Foreign Key)
- LeaveTypeID (Foreign Key)
- StartDate
- EndDate
- TotalDays
- Status (Approved, Rejected, Pending)

**Table 4: Holiday**

- HolidayID (Primary Key)
- HolidayName
- HolidayDate
- Description

This database design enables the employer to keep track of employee information, leave types, leave requests and holiday information. By linking the "Leave" table to the "Employee" table through the "EmployeeID" field, the employer can see all leave requests and their status for each employee. Similarly, by linking the "Leave" table to the "LeaveType" table through the "LeaveTypeID" field, the employer can see the type of leave an employee has requested. Finally, the "Holiday" table can store information about company holidays, such as the holiday date and description, which can be used to plan employee leaves and work schedules.
