Write a security policy outlining the proper use of secure passwords.
**Secure Password Policy**

Purpose:
This policy outlines the requirements for creating and using strong, secure passwords for all systems and applications to safeguard company and user data.

Scope:
This password policy pertains to all employees, contractors, third parties, and others with access to organizational systems or data.

Policy:

1. Password Complexity: Passwords must be at least 8 characters long and contain a combination of lowercase and uppercase letters, numbers, and special characters.

2. Password Expiration: Passwords must be changed every 90 days. Employees must also change their password if they have reason to believe the password has been compromised.

3. Password Reuse: Employees are prohibited from reusing the same password within a 12-month period.

4. Password Sharing: Passwords should never be shared, written down or otherwise exposed. If a password needs to be shared, it must be done through an approved password manager.

5. Default Passwords: Default passwords for new accounts or devices should be changed immediately.

6. Multifactor Authentication: Wherever possible, use multifactor authentication to verify the identity of users when logging in.

7. Reporting: Any employee who suspects that their password has been compromised must report the incident to the IT department immediately.

8. Training: All users must receive regular training on the importance of password security and this policy.

Non-Compliance:
Failure to comply with this password security policy could result in disciplinary action, loss of access to company systems or data, and possibly further legal action.

Conclusion:
It is important to choose secure passwords and to keep them confidential to protect our systems and data against unauthorized access. By following this password security policy, we can ensure that our data remains protected and secure.
