Building a web application using Drupal has both advantages and disadvantages:
 
Advantages:
- Drupal is an open-source content management system (CMS) so it can be used and modified for free.
- This system is highly customizable and is used for website building and web app development, it offers a variety of modules and extensions.
- Drupal is relatively easy to use, does not require additional costs and resources for learning, and has an active community upgrading versions and providing technical support.
- It is designed for managing complex websites and offers robust security and stability, making it perfect for enterprise-level applications.
- It offers a variety of templates that can be used to create engaging websites quickly, which saves time especially in the design process.

Disadvantages:
- While Drupal is relatively easy to use, it still requires some technical knowledge, and it can be challenging for beginners to fully utilize all its advanced features.
- The learning curve can be steep for users who have little to no experience with web development, and it may take longer for them to feel comfortable and confident using it.
- Compared to other CMS, managing and customizing Drupal can be time-consuming, which can be an issue in project management where time is of great importance.
- Drupal-powered websites may experience slower loading times, as all the features can have a negative impact on website performance.
- It requires more resources than other CMS to run effectively, so there may be additional costs on hosting and maintenance.
